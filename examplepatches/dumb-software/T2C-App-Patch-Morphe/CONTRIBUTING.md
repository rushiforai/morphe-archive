# 👋 Contribution guidelines

This document describes how to contribute to T2C App Patches.

## 📖 Resources to help you get started

* [Issues](https://github.com/dumb-software/T2C-API-Documentation/issues) are where we keep track of bugs and feature requests.

## 🙏 Submitting a feature request

Features can be requested by opening an issue using the
[Feature request issue template](https://github.com/dumb-software/T2C-API-Documentation/issues/new?labels=Feature+request&template=feature_request.yml&title=feat%3A+).

## 🐞 Submitting a bug report

If you encounter a bug while using T2C App Patches, open an issue using the
[Bug report issue template](https://github.com/dumb-software/T2C-API-Documentation/issues/new?labels=Bug+report&template=bug_report.yml&title=bug%3A+).

## 📝 How to contribute

1. Before contributing, it is recommended to open an issue to discuss your change with the maintainers.
2. Development happens on the `dev` branch. Fork the repository and create your branch from `dev`.
3. Commit your changes using semantic commit messages.
4. Submit a pull request to the `dev` branch.
5. Our team will review your pull request and provide feedback.

❤️ Thank you for considering contributing to T2C App Patches!
