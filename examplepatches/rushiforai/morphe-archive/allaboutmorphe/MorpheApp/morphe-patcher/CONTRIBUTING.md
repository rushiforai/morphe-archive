# 👋 Contribution guidelines

This document describes how to contribute to Morphe Patcher.

## 📖 Resources to help you get started

- The [documentation](https://github.com/MorpheApp/morphe-patcher/tree/docs/docs) contains the fundamentals
  of Morphe Patcher and how to create patches.
- [Issues](https://github.com/MorpheApp/morphe-patcher/issues) are where we keep track of bugs and feature requests.

## 🙏 Submitting a feature request

Features can be requested by opening an issue using the
[Feature request issue template](https://github.com/MorpheApp/morphe-patcher/issues/new?assignees=&labels=Feature+request&projects=&template=feature_request.yml&title=feat%3A+).

## 🐞 Submitting a bug report

If you encounter a bug while using Morphe Patcher, open an issue using the
[Bug report issue template](https://github.com/Morphe/morphe-patcher/issues/new?assignees=&labels=Bug+report&projects=&template=bug_report.yml&title=bug%3A+).

## 📝 How to contribute

1. Before contributing, it is recommended to open an issue to discuss your change
   with the maintainers of Morphe Patcher. This will help you determine whether your change
   is worth your time to implement it
2. Development happens on the `dev` branch. Fork the repository and create your branch from `dev`
3. Commit your changes
4. Submit a pull request to the `dev` branch of the repository and reference issues
   that your pull request closes in the description of your pull request
5. Our team will review your pull request and provide feedback. Once your pull request is approved,
   it will be merged into the `dev` branch and will be included in the next release of Morphe Patcher

❤️ Thank you for considering contributing to Morphe Patcher.
