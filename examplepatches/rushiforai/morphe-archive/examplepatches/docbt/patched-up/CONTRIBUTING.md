# Contribution guidelines

## Resources

* [Issues](https://github.com/docbt/patched-up/issues) — bug reports and feature requests

## Submitting a feature request

Open an issue using the
[Feature request template](https://github.com/docbt/patched-up/issues/new?labels=Feature+request&template=feature_request.yml&title=feat%3A+).

## Submitting a bug report

Open an issue using the
[Bug report template](https://github.com/docbt/patched-up/issues/new?labels=Bug+report&template=bug_report.yml&title=bug%3A+).

## How to contribute

1. Open an issue to discuss your change before starting
2. Fork the repository and create your branch from `dev`
3. Commit your changes using [semantic commit messages](https://kapeli.com/cheat_sheets/Semantic_Commits.docset/Contents/Resources/Documents/index)
4. Submit a pull request to the `dev` branch and reference any related issues
5. Your pull request will be reviewed and merged once approved

Thank you for contributing!
